## Summary

This tool collects **no personal data**, makes **no network requests** on its own,
requires **no authentication**, and has **no telemetry or analytics of any kind**.
All output is written locally to your own filesystem.

---

## Data collection

| Category | Collected? | Detail |
|----------|-----------|--------|
| User account / identity | **No** | No sign-up, no login, no user ID |
| Usage analytics / telemetry | **No** | No tracking code anywhere in the package |
| API keys / credentials | **No** | Secrets are read from *your* env vars; never logged, sent, or stored by the tool |
| Output files | **Local only** | Written to `tmp/<provider>/<entity>/` under your working directory |
| Audit log | **Local only** | `compliance/audit_log/contributions.jsonl` — written only when you call `log_ai_contribution()` |
| Network requests | **None by default** | The CLI scaffold command and welcome screen make zero network calls |

The `requests` library is bundled as a dep for `BaseRestFetcher` and `BaseGraphQLFetcher`,
but it only makes HTTP calls **to URLs you specify** in your own subclass code.
The library itself does not phone home.

## GDPR (Regulation (EU) 2016/679)

### Role analysis

Under GDPR, roles depend on context:

| Actor | GDPR role |
|-------|-----------|
| **You** (the developer or organisation running this tool) | **Data controller** for any personal data your ingest fetchers retrieve from APIs or databases |
| **Aevoxis Solutions** | **Not a processor** — we never receive, process, or have access to any data your tool handles |
| **pr-automation-agent** | Infrastructure / development tooling — not a data processor under GDPR because it never processes data on behalf of a controller; it runs entirely within your environment |

### Obligations for you as controller

If your ingest files retrieve records that contain personal data (e.g. user profiles,
email addresses, order histories) you must:

- Have a lawful basis for the processing (Art. 6)
- Implement appropriate technical and organisational measures (Art. 32)
- Respect data subject rights (Art. 15–22)
- Maintain a record of processing activities (Art. 30)
- Store personal data only as long as necessary (Art. 5(1)(e))

The tool itself does not impose any retention period on its output files —
`tmp/` is whatever you configure it to be. You are responsible for managing
the lifecycle of any personal data in those files.

### Audit log data

The optional audit log (`compliance/audit_log/contributions.jsonl`) stores:

```json
{
  "timestamp": "2026-06-30T14:22:00Z",
  "file_path": "ingest/rest/stripe/invoices_asset.py",
  "ai_model": "GitHub Copilot",
  "human_reviewer": "@yourhandle",
  "pr_number": "123"
}
```

`@yourhandle` is a GitHub username — this may be personal data under GDPR.
You control this file. If you process EU residents' GitHub handles, this file
is part of your record of processing activities (Art. 30).

## EU AI Act (Regulation (EU) 2024/1689)

### Risk classification

**Limited Risk** — not Annex III (high-risk), not Annex I (prohibited).

The tool is a developer productivity aid that generates boilerplate code.
It does not make consequential decisions about natural persons, does not
process biometric data, and does not influence access to essential services.

### Obligations and how they are met

| Article | Obligation | Implementation in this package |
|---------|-----------|-------------------------------|
| Art. 50 | Disclose AI-generated content when interacting with natural persons | PR template checkbox + `ai-generated` GitHub label on every AI-scaffolded PR |
| Art. 52 | Transparency for AI-generated code | Art. 52 header auto-inserted in every scaffolded file |
| Art. 53/56 | GPAI deployer transparency notice | `compliance/AI_TRANSPARENCY_NOTICE.md` |
| Art. 14 (practice) | Meaningful human oversight | `compliance/HUMAN_OVERSIGHT_POLICY.md` — requires human review before merge |

The Art. 52 header inserted into every generated file reads:

```
# EU AI Act Art. 52 — AI-Generated Content
# Generated by: pr-agent scaffold   Date: YYYY-MM-DD
# Human reviewer required before merging (see compliance/HUMAN_OVERSIGHT_POLICY.md)
```

### Audit trail

`log_ai_contribution()` appends to a local JSONL file. This is your
Art. 30 / Art. 14 audit trail for AI-generated code contributions.
It is never transmitted anywhere.

## NIS2 / cybersecurity

This tool has no network-facing surface and does not run as a service.
It is a local CLI that writes files to your filesystem.
No NIS2 obligations apply to the tool itself.

If your organisation is subject to NIS2 as an essential or important entity,
you are responsible for securing the environments where the tool runs and the
secrets it reads.

## Data retention and deletion

All data the tool writes is on your own machine:

| Location | Contents | How to delete |
|----------|----------|---------------|
| `tmp/` | JSON / Parquet ingest output | `rm -rf tmp/` |
| `compliance/audit_log/contributions.jsonl` | AI contribution records | Delete the file or specific lines |

There is no data to request deletion of from Aevoxis — we have none.
